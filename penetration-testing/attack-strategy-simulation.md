# Security Decision-Making and Risk Management Strategy

**Prepared by:** James Wigfall  

---

## Overview

This project outlines a series of security decisions made during the development and deployment of an application. The focus is on balancing cost, risk, and operational efficiency while maintaining a strong security posture.

---

## Security Strategy Decisions

### Third-Party Security Assessment

A third-party audit team was selected to evaluate the application’s security.

- Enables independent validation of security posture  
- Provides expert insight into vulnerabilities  
- Supports faster identification of critical risks  

---

### Vendor Selection

A freelance security team was chosen based on:

- Cost efficiency  
- Technical capability  
- Faster turnaround time  

This approach balanced quality and budget constraints without overcommitting resources.

---

### Internal Security Controls

Existing infrastructure was leveraged before introducing new tools.

- Active Directory used to enforce strict access control policies  
- Focus on strengthening foundational security configurations  
- Reduced unnecessary spending on additional solutions  

---

### Data Protection Strategy

All intellectual property was encrypted to reduce risk in case of compromise.

- Protects sensitive data from unauthorized access  
- Ensures data remains unusable if breached  
- Aligns with data protection best practices  

---

### Threat Detection and Monitoring

Investment in breach detection technology was prioritized.

- Enables real-time monitoring of system activity  
- Improves detection of suspicious behavior  
- Supports incident response and forensic analysis  

---

### Controlled Testing Environment

External users were only allowed to interact with the system in a controlled environment.

- Reduces exposure to exploitation  
- Limits risk of unintended vulnerabilities being discovered  
- Maintains system integrity during testing  

---

### External Communication Strategy

A transparent but controlled communication approach was used.

- Sensitive system details were not publicly disclosed  
- Maintained trust while protecting critical information  
- Reduced risk of information leakage  

---

### Incident Monitoring

Security logs were actively reviewed to identify potential issues.

- Regular log analysis improves detection capabilities  
- Supports proactive threat identification  
- Enhances overall system visibility  

---

## Lessons Learned

A balanced approach to security is essential for effective risk management.

Key takeaways include:

- Security should combine both internal and external strategies  
- Cost-effective decisions can still maintain strong security  
- No system is fully secure—monitoring and detection are critical  
- Encryption is essential for protecting sensitive data  
- Transparency must be balanced with operational security  

---

## Conclusion

This project demonstrates the importance of strategic decision-making in cybersecurity. By combining cost management, security controls, and proactive monitoring, organizations can strengthen their defenses while maintaining operational efficiency.

A layered security approach, supported by continuous monitoring and regular assessments, provides the best defense against evolving threats.
